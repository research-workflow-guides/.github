# Contributing

Thanks for helping improve this project.

## Good Contributions

- Fix unclear or outdated documentation.
- Add examples that make a workflow easier to follow.
- Report broken links, rendering issues, or confusing instructions.
- Keep changes focused and easy to review.

## Pull Requests

Before opening a pull request:

1. Check whether an existing issue or pull request already covers the change.
2. Keep the change scoped to one topic.
3. Explain what changed and why.
4. Include screenshots for visual changes when useful.

## Writing Style

- Prefer direct, practical language.
- Avoid unexplained jargon.
- Use examples when they reduce ambiguity.
- Keep headings and filenames descriptive.
